## [[科技] Crustc：将整个 Rust 编译器翻译为 C 语言](https://github.com/FractalFir/crustc) ⭐️ 8.0/10

**发生了什么**: 一个名为 crustc 的项目已将整个 rustc 编译器翻译为 C 语言，从而可以通过 C 编译器来编译 Rust 代码。这是已知的第 14 次此类转译尝试。

**投资影响**: 这可能使 Rust 能够在没有 LLVM 或 GCC 支持的平台上运行，解决了需要 Rust 编译器来构建自身的引导问题。它还为通过多样化双编译（DDC）验证来检测编译器后门提供了可能性。

**关键信息**: 该项目由 FractalFir 开发，已持续三年。它将 rustc 转译为 C 语言，允许 GCC 优化生成的代码，这可能比直接针对 LLVM IR 更实用。

hackernews · Philpax · 7月2日 22:57 · [社区讨论](https://news.ycombinator.com/item?id=48768464)

**背景**: Rust 的主要编译器 rustc 基于 LLVM，这限制了其对 LLVM 目标平台的支持。从源码引导 Rust 需要一个现有的 Rust 编译器，这为新平台带来了先有鸡还是先有蛋的问题。将 rustc 转译为 C 语言打破了这种依赖，因为 C 编译器几乎在所有平台上都可用。

**社区讨论**: 社区表达了浓厚兴趣，评论称赞了作者的奉献精神，并指出了 DDC 验证的潜力。一些人讨论了 LLVM C 后端作为替代方案，但指出它目前并未维护。一位评论者幽默地提到了搅拌机事故，但总体情绪是积极且对实现细节充满好奇的。

**标签**: `#rust`, `#compiler`, `#transpilation`, `#bootstrapping`, `#systems programming`
